Part 1 · HTML

文档结构——网页的骨架

完整 HTML 骨架
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>我的网页</title>
</head>
<body>
  <!-- 页面内容写这里 -->
</body>
</html>

<!DOCTYPE html>

告诉浏览器"我是 HTML5 文档"，避免怪异渲染模式。

<html>

根元素，整个页面的一切都在它里面。lang 属性帮助搜索引擎和屏幕阅读器识别语言。

<head>

元数据区——字符编码、标题、CSS 引用、SEO 信息。用户看不到这些内容。

<body>

可见内容区——你看到的一切文字、图片、按钮，都在 body 里。

浏览器解析 HTML 后生成 DOM 树（Document Object Model），这是一切的起点——CSS 和 JS 都基于它工作。
Part 1 · HTML

元素三分类

🧱

块级元素

Block-level

独占一行，像段落一样铺满宽度。可以设宽高。

<div> <p> <h1> <ul>
🔗

内联元素

Inline

和文字同行，只占内容宽度。设宽高无效。

<span> <a> <strong> <em>
👻

空元素

Void / Self-closing

没有子内容，不需要闭合标签。功能全靠属性携带。

<img> <br> <input> <meta>
CSS 的 display 属性可以强制改变元素的渲染类型——块级可以变内联，内联也可以变块级。
Part 1 · HTML

核心标签速查表

标签用途关键属性小提示
<h1>~<h6>标题层级—每页只一个 h1，SEO 权重递减
<p>段落—自动有上下 margin
<a>超链接href, targettarget="_blank" 新标签页打开
<img>图片src, altalt 是无障碍必填项
<ul><li>无序列表—ol 是有序列表
<div>通用容器—无语义，纯布局用
<span>行内容器—给文字片段加样式
<main>主内容区—每页只能一个
<article>独立文章—可单独分发的内容块
<section>内容分区—应有标题 heading
<nav>导航区—通常含 a 链接列表
<header>头部区—可复用于 article 内
<footer>底部区—版权、联系信息等
语义标签不只是"好看"——它影响 SEO 排名和无障碍访问（屏幕阅读器据此导航），div 没有这些能力。

表单系统——数据入口

完整表单代码
<form action="/submit"
      method="POST">

  <!-- 文本输入 -->
  <label>姓名</label>
  <input type="text"
         name="username"
         required>

  <!-- 邮箱输入 -->
  <input type="email"
         name="email">

  <!-- 提交按钮 -->
  <button type="submit">
    提交
  </button>

</form>

input type 速查

text 单行文本
email 邮箱验证
password 密码遮罩
number 数字输入
checkbox 多选框
radio 单选组
file 文件上传
range 滑块

action = 数据提交到哪个 URL

method = GET（URL 可见）或 POST（请求体传输）

name = 每个字段的标识名，后端据此取值

表单是前端收集数据、传给后端的标准通道。method=POST 表示数据通过请求体提交到服务器，比 GET 更安全（不会暴露在 URL 里）。

Part 2 · CSS

CSS 三要素 + 优先级

解剖一行 CSS
h1 {
  color: #8b5cf6;
  font-size: 2rem;
}
选择器 h1 属性 color 值 #8b5cf6
三种选择器
/* 标签选择器 */
p { color: gray; }

/* 类选择器 */
.card { padding: 16px; }

/* ID 选择器 */
#hero { font-size: 3rem; }

三种写法

行内样式 style="..." — 最高优先级

style 标签 <style> — 页面内部

外部文件 <link> — 最推荐 ✅

优先级规则

行内 > ID > 类 > 标签

同优先级时，后写的覆盖先写的（就近原则）

为什么推荐外部文件？

多个页面可以复用同一份 CSS，浏览器还能缓存它——加载更快，维护更方便。

这叫 Specificity（特异性），浏览器按特异性分值决定哪条规则生效——ID(100) > Class(10) > Tag(1)，分值高的覆盖低的。

盒模型——每个元素都是个盒子

margin
border
padding
content

🟧 Margin 外边距

盒子与外界的距离。可以是负值。auto 可实现水平居中。

🟪 Border 边框

盒子的边线。有三个子属性：width / style / color。

🟦 Padding 内边距

内容与边框之间的空间。不可以是负值。

⬜ Content 内容

文字、图片等实际内容区域。width/height 默认控制这里。

默认盒模型（content-box）的 width 不含 padding 和 border，加上 box-sizing: border-box 后 width 包含两者，更符合直觉——全局设置 * { box-sizing: border-box } 是最佳实践。

Part 3 · JavaScript

四个积木——JS 的基础组件

📦

变量 Variable

像贴了标签的盒子——存东西，随时取。

let name = "小明";  // 可变
const PI = 3.14;   // 不可变

let/const 是块级作用域声明，var 是函数作用域（已不推荐）。

🔧

函数 Function

像菜谱——把步骤包起来，需要时调用。

function greet(name) {
  return `你好，${name}！`;
}
greet("小明"); // 调用它

箭头函数 const greet = (name) => ... 是更简短的写法。

🔀

条件 Conditional

像岔路口——满足条件走这边，否则走那边。

if (age >= 18) {
  // 成年人逻辑
} else {
  // 未成年人逻辑
}

条件表达式会被隐式转换为布尔值（truthy / falsy）。

🔄

循环 Loop

像唱片机——重复执行，直到满足退出条件。

for (let i = 0; i < 5; i++) {
  // 执行 5 次
}

for...of 遍历可迭代对象，for...in 遍历对象键名。

Part 3 · JavaScript

DOM 操控——让 JS 读写页面

DOM 树可视化
document └── <html> ├── <head> │ └── <title> └── <body> ├── <h1> ├── <p> └── <button>

DOM 树：浏览器把 HTML 解析成的对象结构，每个标签是一个节点（Node）。

① 找元素

const el = document.querySelector('.card');
// 返回第一个匹配的元素

② 改内容

el.textContent = '新内容';  // 纯文本
el.innerHTML = '<strong>加粗</strong>';

③ 改样式

el.style.color = '#3b82f6';
el.classList.add('active');
// classList 比 style 更灵活
DOM（Document Object Model）是浏览器暴露给 JS 的接口，让代码能读写页面上的任何元素——本质上是在操作一棵由节点组成的树。

常用事件

事件触发时机
click鼠标点击
keydown键盘按下
input输入框内容变化
submit表单提交
DOMContentLoadedDOM 加载完毕
完整事件监听示例
const btn = document.querySelector('#myBtn');

// 添加点击监听
btn.addEventListener('click', function() {
  // 点击时执行这段代码
  document.querySelector('#output')
    .textContent = '按钮被点了！';
});
addEventListener 是标准的事件绑定方式。事件会从目标元素向上冒泡（bubbling），可以在祖先元素上统一监听——这叫事件委托。

综合 Demo——待办清单

完整代码（HTML + JS）
<!-- HTML 结构 -->
<input id="todo-input"
       placeholder="输入任务，回车添加">
<ul id="todo-list"></ul>

<!-- JavaScript 逻辑 -->
const input = document.querySelector(
  '#todo-input'
);
const list = document.querySelector(
  '#todo-list'
);

input.addEventListener('keydown',
  function(e) {
    if (e.key === 'Enter'
        && input.value) {
      if (list.children.length >= 6) {
        return alert('最多6条');
      }
      const li = document.createElement('li');
      li.textContent = input.value;
      li.addEventListener('click',
        () => li.classList.toggle('done'));
      list.appendChild(li);
      input.value = '';
    }
  }
);

🧪 试一试：输入任务，回车添加

    点击条目可以划掉 ✅

    这就是一个完整的交互功能：DOM 操作（创建元素）+ 事件监听（keydown/click）+ 条件判断（Enter 键 / 6 条限制）。
